# Kairos Build Spec v2 Implementation Notes

Date: 2026-06-02

## What Was Incorporated

Claude's spec was folded into Kairos with Vercel/manual deployment constraints in mind. Streamlit deployment guidance was ignored as requested. Automation is daily-safe for Vercel Hobby; no sub-daily cron assumption is required.

Implemented additions:

- Data-truth verification protocol script.
- NSE factor CSV validator and manual CSV upload support.
- Risk-free-rate cascade with FBIL -> CCIL -> versioned fallback CSV.
- Regression now fails with `insufficient_rfr` when RFR is missing rather than using zero/constant silently.
- Portable Alpha Score table and computation scaffold.
- Factor-matched peer DiD engine.
- Transition Impact Forecast engine with 12M/24M NAV-impact percentiles.
- Daily-safe monitor agent.
- Agent run logging.
- Demo alert/report generator writing HTML transition briefs and `alert_log` rows.
- Vercel-friendly Python API endpoints.
- Daily Vercel cron endpoint only: `/api/cron_daily` at `0 3 * * *`.
- DuckDB sidecar retained for analytical scans.

## Intentional Deviations From Claude Spec

- No sub-daily scheduler. Vercel Hobby practicality means daily cron only.
- No Streamlit refactor for deployment. The project now has Vercel API handlers; UI/deployment can be handled separately.
- No Claude provider added. Existing NVIDIA-first LLM layer remains to avoid adding another dependency/key path.
- No live SMTP/Twilio delivery yet. Alert rows and HTML transition reports are generated; actual delivery can be connected later.
- No WeasyPrint dependency yet. Transition briefs are HTML files for portability; PDF rendering can be added once deployment environment is chosen.
- No forced 50 verified transitions generated by code. The system provides templates, verification, and seed scaffolding; real verification remains a human/data task.
